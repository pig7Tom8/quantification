# Phase 2：市场状态与拥挤度

## 阶段目标

避免系统在市场风险较高或题材过热时盲目选股、盲目开仓，为评分和交易动作增加环境约束。

## 对应原方案内容

- 市场状态因子
- 系统性风险
- 题材拥挤度判断
- 综合评分与评级中的市场状态约束
- Phase 2：市场状态 + 拥挤度，1 ~ 2 周

## 本阶段范围

```text
- 实现市场状态判断
- 实现市场状态对仓位的约束
- 实现题材拥挤度轻量判断
- 实现拥挤度对评级的调整
```

## 市场状态

```text
market_state = 进攻 / 震荡 / 防守
```

### 市场状态用途

```text
- 控制是否允许开新仓
- 控制总仓位上限
- 控制 S/A 级股票是否参与虚拟买入
```

### 动作约束

```text
进攻：S / A 可参与虚拟买入
震荡：S 可虚拟买入，A 只观察
防守：所有评级只观察，不开新仓
```

## 题材拥挤度

### 判断目的

```text
- 判断题材是否过热
- 判断是否存在高位接力风险
- 决定是否降低评级或禁止追高
```

### 拥挤度分级

```text
低拥挤：可以正常参与
中拥挤：降低评级或降低仓位
高拥挤：禁止追高，只观察
极高拥挤：不新开仓，已有持仓加强止盈
```

### 评级影响

```text
高拥挤：
- S → A
- A → B / 观察

极高拥挤：
- S / A 均不触发买入
- 只进入观察池
```
