# 开发总览

## 目的

本目录在不改变原方案内容的前提下，将《股票量化系统 MVP 修订版方案》按后续开发步骤拆分为可执行文档，方便逐阶段推进。

## 拆分方式

共 7 份文档：

1. 本文档：开发总览
2. Phase 0：项目骨架
3. Phase 1：因子评分
4. Phase 2：市场状态与拥挤度
5. Phase 3：资讯抓取与 LLM 结构化
6. Phase 4：虚拟跑盘与基准对比
7. Phase 5：策略快照与飞书推送

## 对应原方案阶段

- Phase 0：项目骨架，1 周
- Phase 1：因子评分，2 周
- Phase 2：市场状态 + 拥挤度，1 ~ 2 周
- Phase 3：资讯抓取 + LLM 结构化，2 周
- Phase 4：虚拟跑盘 + 基准对比，2 周
- Phase 5：strategy_daily_snapshot + 飞书推送，1 周

## 开发顺序

```text
Phase 0
→ Phase 1
→ Phase 2
→ Phase 3
→ Phase 4
→ Phase 5
```

## 每阶段产物关系

```text
Phase 0：先把数据、库表、调度、日报骨架跑通
Phase 1：让系统先能打分和评级
Phase 2：给评分增加市场环境和题材拥挤度约束
Phase 3：把资讯转成结构化事件和资讯分
Phase 4：验证策略是否有效，是否跑赢基准
Phase 5：沉淀每日快照，并自动推送日报
```

## 每日最小闭环

```text
1. 更新行情数据
2. 更新财务数据
3. 抓取公告和资讯
4. LLM 提取事件信息
5. 规则引擎计算资讯分
6. 计算市场状态
7. 计算题材拥挤度
8. 计算个股多因子评分
9. 生成 S/A/B/C/D/R 评级
10. 策略虚拟账户调仓
11. 基准账户同步更新
12. 计算超额收益、最大回撤、胜率
13. 写入 strategy_daily_snapshot
14. 生成 Markdown / CSV 报告
15. 飞书机器人推送日报
```

## 最终 MVP 版本定义

```text
Python 定时任务 + 数据库 + 多因子评分 + LLM 资讯结构化 + 市场状态控制 + 拥挤度判断 + 虚拟跑盘 + 基准对比 + 飞书日报推送
```
