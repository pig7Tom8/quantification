# Phase 3：资讯抓取与 LLM 结构化

## 阶段目标

让系统知道“为什么涨”，但不让 LLM 直接决定买卖，而是将非结构化资讯转成规则可计算的事件和分数。

## 对应原方案内容

- 数据源设计中的资讯数据
- 资讯因子
- LLM 的职责与边界
- 资讯评分规则
- 置信度处理
- Phase 3：资讯抓取 + LLM 结构化，2 周

## 本阶段范围

```text
- 抓公告
- 抓新闻
- 识别股票代码
- LLM 提取事件类型
- LLM 提取关键词
- LLM 输出置信度
- 规则引擎计算资讯分
- 写入 news_events
```

## LLM 的职责

```text
1. 提取关键词
2. 识别事件类型
3. 提取涉及公司、行业、产品、金额、时间
4. 判断事件方向：正面 / 中性 / 负面
5. 输出置信度
6. 生成一句话摘要
```

## LLM 不做的事

```text
- 不直接判断是否买入
- 不直接判断是否卖出
- 不直接决定最终评分
- 不直接生成 0/1 决策
```

## 资讯分规则

```text
资讯分：-15 ~ +15
```

## 置信度处理

```text
高置信度：正常计分
中置信度：分数 * 0.5
低置信度：不计入评分，只进入观察
```

## 需要落地的模块

```text
backend/

```

## 本阶段使用的数据表

- news_events
- factor_score

## 本阶段完成标志

```text
1. 每日可抓取公告和新闻
2. 可识别股票代码和事件类型
3. LLM 可输出摘要、方向、置信度
4. 规则引擎可生成 NewsScore
5. 资讯结果可写入 news_events
```

## 交付物

```text
每只股票都有资讯摘要、事件类型、置信度和资讯分。
```

Phase 4 会把评级、市场状态、拥挤度和资讯分统一用于虚拟买卖，开始验证策略是否真正有效。
