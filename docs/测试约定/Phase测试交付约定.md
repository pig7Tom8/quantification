# Phase 测试交付约定

## 目的

在不改变现有方案的前提下，为后续每个开发阶段增加统一测试交付要求，确保每完成一个 Phase，都有可复用、可复查、可手动执行的测试用例。

## 统一要求

每完成一个开发阶段，必须同步完成以下事项：

```text
1. 生成该阶段对应的测试用例文档
2. 按测试用例逐条执行所有 case
3. 修复未通过项，直到全部 case 跑通
4. 输出测试结果摘要
5. 交给你按同一份测试用例复测
```

## 执行顺序

```text
阶段开发完成
→ 生成测试用例文档
→ 我执行全部测试 case
→ 修复问题
→ 全部 case 跑通
→ 交付给你复测
```

## 测试用例文档要求

每个 Phase 的测试用例文档统一包含以下部分：

```text
1. 测试目标
2. 测试范围
3. 前置条件
4. 测试数据准备
5. 测试步骤
6. 预期结果
7. 异常场景
8. 回归检查项
9. 测试结果记录
```

## Case 设计要求

每个 Phase 的测试 case 至少覆盖：

```text
- 正常流程 case
- 边界条件 case
- 异常输入 case
- 空数据或缺失数据 case
- 回归 case
```

## 交付标准

只有当以下条件全部满足，当前 Phase 才算完成：

```text
1. 功能开发完成
2. 测试用例文档完成
3. 我已按文档执行全部 case
4. 所有 case 均通过
5. 已输出测试结果摘要
```

## 后续执行方式

后面每进入一个新 Phase，我会同步输出两份交付：

```text
- 开发实现
- 对应 Phase 测试用例文档
```

并且我会先自己把所有 case 跑通，再交给你复跑。
