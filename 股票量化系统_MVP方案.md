# 股票量化系统 MVP 方案

## 0. 定位

目标不是一开始做“全自动交易系统”，而是先做一个：

> **能自动选股、打分评级、抓资讯、虚拟跑盘、每日推送，并持续验证策略有效性的个人量化决策系统。**

第一阶段不直接实盘自动下单，避免风控、券商接口、合规和策略未验证的问题。

参考成熟系统的思路：

- **vn.py / VeighNa**：偏完整量化交易平台，包含数据、回测、实盘交易等能力，适合后期参考架构。
- **Backtrader**：强调策略、指标、回测、分析器，适合 MVP 阶段做策略验证。
- **Freqtrade**：有回测、资金管理、策略优化、WebUI / Telegram 控制等，推送和策略迭代思路值得借鉴。
- **Qlib**：适合后期做 AI / 因子研究 / 机器学习选股，但 MVP 阶段不建议一开始重度 ML 化。

## 1. MVP 核心功能

### 1.1 股票池管理

#### 初始股票池

先不要覆盖全市场，建议从：

- 沪深 A 股主板 + 创业板 + 科创板
- 剔除 ST、退市风险、长期停牌
- 剔除上市不足 120 个交易日的新股
- 剔除日均成交额过低的股票，例如近 20 日平均成交额 < 5000 万

#### 股票池分类

```text
全市场股票池
├── 可交易股票池
│   ├── 主线题材池
│   ├── 业绩增长池
│   ├── 趋势强势池
│   ├── 低位反转池
│   └── 风险观察池
```

## 2. 选股策略设计

MVP 阶段建议先做 **多因子打分 + 规则过滤**，不要一开始就搞复杂机器学习。

### 2.1 选股总逻辑

```text
基础过滤
→ 行情因子
→ 资金因子
→ 财务因子
→ 题材资讯因子
→ 风险因子
→ 综合评分
→ 股票评级
→ 虚拟买入/观察
```

## 3. 因子体系

### 3.1 技术趋势因子，权重 30%

| 因子 | 说明 | 加分逻辑 |
|---|---|---|
| MA5 > MA10 > MA20 | 短期均线多头 | 加分 |
| 股价站上 MA20 | 趋势转强 | 加分 |
| 20 日涨幅 | 判断强度 | 适度加分，过高扣分 |
| 60 日新高 | 趋势突破 | 加分 |
| 放量上涨 | 量价配合 | 加分 |
| 缩量回踩 | 强势整理 | 加分 |

建议规则：

```text
趋势分 = 0 ~ 30

+5  股价 > MA20
+5  MA5 > MA10 > MA20
+5  最近 20 日涨幅在 5% ~ 35%
+5  最近 10 日有放量阳线
+5  距离 60 日新高小于 5%
+5  回撤小于 15%
```

扣分：

```text
-5  跌破 MA20
-5  最近 20 日涨幅 > 60%，短线过热
-5  最近 5 日放量下跌
-10 股价跌破 MA60
```

---

### 3.2 资金强度因子，权重 25%

| 因子 | 说明 |
|---|---|
| 成交额排名 | 判断资金关注度 |
| 换手率 | 判断活跃度 |
| 主力净流入 | 可参考，但不要过度依赖 |
| 北向资金 | 适合大票 |
| 龙虎榜 | 适合短线题材股 |

建议规则：

```text
资金分 = 0 ~ 25

+5  近 5 日成交额持续放大
+5  今日成交额大于近 20 日均值 1.5 倍
+5  换手率在 3% ~ 15%
+5  近 3 日资金净流入为正
+5  所属板块成交额排名靠前
```

扣分：

```text
-5  高位爆量长上影
-5  连续 3 日资金流出
-5  换手率过高，比如 > 30%
```

---

### 3.3 基本面因子，权重 20%

MVP 不需要做太复杂，先抓关键财务指标。

| 因子 | 说明 |
|---|---|
| 营收同比增长 | 判断成长性 |
| 净利润同比增长 | 判断盈利能力 |
| 毛利率变化 | 判断竞争力 |
| 资产负债率 | 风险控制 |
| ROE | 盈利质量 |

建议规则：

```text
基本面分 = 0 ~ 20

+5  营收同比增长 > 10%
+5  净利润同比增长 > 10%
+5  ROE > 8%
+3  毛利率稳定或提升
+2  资产负债率不过高
```

扣分：

```text
-5  净利润连续下滑
-5  扣非净利润为负
-5  商誉占比过高
-5  资产负债率明显高于行业平均
```

---

### 3.4 题材资讯因子，权重 15%

这是系统的重点，因为你平时会关注 AI 算力、PCB、Kimi、资源、灵工平台、电力交易等题材。

资讯分不建议完全交给大模型主观判断，而是：

```text
关键词匹配 + 事件分类 + 情绪判断 + 可信源权重
```

#### 资讯类型

```text
政策利好
产业趋势
订单合同
业绩预增
涨价逻辑
技术突破
机构调研
高管增持
减持风险
监管风险
澄清公告
业绩暴雷
```

#### 资讯评分

```text
资讯分 = -15 ~ +15

+5  公司公告出现订单、业绩预增、回购、增持
+5  所属行业出现政策或产业利好
+3  多家媒体报道同一正面事件
+2  机构调研热度上升

-5  减持、监管问询、诉讼
-5  业绩预亏、商誉减值
-3  高位澄清风险
-3  负面舆情增多
```

---

### 3.5 风险因子，权重 -20 到 0

风险因子不是加分，而是扣分。

```text
风险分 = 0 ~ -20

-5   ST / 退市风险 / 重大诉讼
-5   近 20 日涨幅过高且无业绩支撑
-5   大股东减持
-5   连续跌破关键均线
-5   业绩亏损扩大
-5   换手率异常过高
-10  监管处罚 / 财务造假风险
```

---

## 4. 综合评分与评级

### 4.1 总分公式

```text
总分 = 趋势分 * 30%
     + 资金分 * 25%
     + 基本面分 * 20%
     + 资讯分 * 15%
     + 风险调整分
```

为了工程实现简单，可以统一换算成 100 分制。

```text
Score = TrendScore + MoneyScore + FundamentalScore + NewsScore + RiskScore
```

### 4.2 评级体系

| 分数 | 评级 | 含义 | 操作建议 |
|---:|---|---|---|
| 85 - 100 | S | 强趋势 + 强资金 + 强题材 | 重点观察 / 虚拟买入 |
| 75 - 84 | A | 有较强机会 | 加入核心观察池 |
| 65 - 74 | B | 有潜力但确认不足 | 观察 |
| 50 - 64 | C | 普通 | 不操作 |
| < 50 | D | 弱势或风险较大 | 剔除 |
| 风险触发 | R | 风险票 | 禁止买入 |

### 5.2 资讯抓取模块

#### 数据源

```text
东方财富公告
巨潮资讯
交易所公告
财联社
证券时报
中国证券报
上海证券报
公司官网
互动易 / 上证 e 互动
雪球 / 股吧，低权重，仅做情绪参考
```

#### 资讯处理流程

```text
抓取原文
→ 清洗标题和正文
→ 识别股票代码
→ 判断事件类型
→ 判断正负面
→ 生成摘要
→ 写入 news_events 表
→ 更新股票资讯分
```

#### 大模型在这里的作用

```text
1. 判断新闻属于什么事件
2. 判断利好/利空/中性
3. 提取影响对象
4. 生成一句话摘要
5. 判断是否需要推送
```

但不要让 LLM 直接决定买卖。

---

### 5.3 策略引擎模块

MVP 先支持 3 类策略。

#### 策略 1：趋势强势股策略

适合找“浪潮信息、工业富联、PCB、算力、资源股”这种主线趋势票。

```text
入选条件：
- 股价 > MA20
- MA5 > MA10 > MA20
- 近 20 日涨幅 5% ~ 40%
- 近 5 日成交额放大
- 所属板块强度排名前 20%
- 无重大负面公告
```

#### 策略 2：低位放量启动策略

适合找刚启动的题材股。

```text
入选条件：
- 近 60 日涨幅 < 30%
- 今日涨幅 > 3%
- 今日成交额 > 近 20 日均值 2 倍
- 股价突破 MA20
- 资讯或板块存在催化
```

#### 策略 3：业绩 + 资金共振策略

适合中线。

```text
入选条件：
- 营收同比增长 > 10%
- 净利润同比增长 > 10%
- ROE > 8%
- 股价趋势不破 MA60
- 近 20 日资金净流入为正
```

## 6. 虚拟跑盘模块

MVP 必须做虚拟跑盘，这比回测更贴近真实使用。

### 6.1 虚拟账户

```text
初始资金：1000000
单票最大仓位：10%
单行业最大仓位：30%
最大持仓数量：10 只
单票最大亏损：-8%
组合最大回撤：-15%
```

### 6.2 虚拟买入规则

```text
S 级股票：
- 次日开盘虚拟买入
- 仓位 8% ~ 10%

A 级股票：
- 次日开盘虚拟买入
- 仓位 5%

B 级股票：
- 不买入，只观察
```

### 6.3 卖出规则

```text
止损：
- 跌破买入价 8%
- 或跌破 MA20
- 或出现重大负面资讯

止盈：
- 盈利 15% 后上移止损
- 盈利 25% 后减半
- 趋势未破则保留底仓

调仓：
- 评分跌破 65，减仓
- 评分跌破 55，清仓
- 出现 R 级风险，清仓
```

## 11. 开发阶段规划

### Phase 0：准备阶段，1 周

目标：搭好骨架。

```text
- 明确股票市场范围
- 确定数据源
- 建 PostgreSQL 表
- 搭 FastAPI
- 跑通定时任务
- 拉取 A 股基础数据
- 拉取日 K 数据
```

交付物：

```text
能每天自动更新股票基础数据和行情数据。
```

---

### Phase 1：选股评分 MVP，2 周

目标：先让系统能每日选股和评级。

```text
- 实现趋势因子
- 实现资金因子
- 实现基础财务因子
- 实现风险过滤
- 实现综合评分
- 实现 S/A/B/C/D 评级
- 输出每日选股列表
```

交付物：

```text
每天收盘后生成：
- Top 50 股票
- S 级股票
- A 级股票
- 风险剔除股票
```

---

### Phase 2：资讯抓取 + AI 摘要，2 周

目标：让系统知道“为什么涨”。

```text
- 抓公告
- 抓新闻
- 抓行业资讯
- 识别股票代码
- 识别事件类型
- 判断利好/利空/中性
- 生成摘要
- 影响资讯分
```

交付物：

```text
每只股票都有资讯摘要和资讯分。
```

---

### Phase 3：虚拟跑盘，2 周

目标：验证策略是否真的赚钱。

```text
- 建虚拟账户

- 计算收益率
- 计算最大回撤
- 计算胜率
- 计算盈亏比
```

交付物：

```text
系统每天根据评分自动虚拟买卖，并生成收益曲线。
```

---

### Phase 4：推送系统，1 周

## 13. MVP 优先级

### 必须做

```text
股票池
行情采集
基础因子
评分评级
选股列表
资讯抓取
AI 资讯摘要
虚拟跑盘
每日推送
```

### 可以晚点做

```text
分钟级行情
复杂回测
机器学习
自动下单
WebSocket 实时行情
多账户管理
高频交易
```

### 不建议 MVP 做

```text
实盘自动交易
复杂深度学习
盘口高频策略
全市场分钟级实时扫描
过度依赖 LLM 买卖决策
```

---

## 14. 推荐 MVP 最小闭环

每天收盘后：

```text
1. 更新行情数据
2. 更新财务数据
3. 抓取公告和资讯
4. 计算因子分
5. 生成股票评级
6. 选出 Top 30
7. 虚拟账户自动调仓
8. 推送收盘报告
```

第二天：

```text
1. 观察评分变化
2. 记录虚拟买卖结果
3. 复盘策略是否有效
4. 调整因子权重
```

---

## 15. 第一版推荐策略组合

MVP 第一版先做这 4 个策略：

```text
1. 趋势强势策略
2. 低位放量启动策略
3. 业绩资金共振策略
4. 风险剔除策略
```

不要一开始做太多策略，否则很难判断到底哪个策略有效。

---

## 16. 一句话架构总结

```text
这是一个以“多因子评分 + 资讯事件分析 + 虚拟跑盘验证 + 每日推送”为核心的 A 股量化选股 MVP。
```

重点不是预测每只股票涨跌，而是建立一套稳定流程：

```text
发现机会 → 给出理由 → 量化评分 → 虚拟验证 → 复盘优化
```

---

## 17. 建议结论

第一版不要做得像 vn.py 那么重，也不要一上来做 Qlib 那种 AI 研究平台。

更适合先做一个：

> **量化选股 + AI 资讯分析 + 虚拟跑盘的轻量系统**

这样最快能跑起来，也最适合日常看股票、看题材、问买卖点的使用习惯。
